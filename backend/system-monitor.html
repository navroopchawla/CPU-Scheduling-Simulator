How CPU usage is managed in practice

1. OS scheduler (kernel)

Windows/Linux/macOS pick which process runs on each core — like your Simulator’s FCFS, RR, or Priority, but with more rules and preemption.

2. Process priority

Important tasks (system, foreground apps) get higher priority. High CPU from one app can starve others — same idea as Priority scheduling starvation.

3. Multicore & load balance

Work is spread across cores (see per-core bars). Uneven bars mean some cores are busier — the OS tries to balance load.

4. What you can do

  • Close unused apps and browser tabs
  • Check Task Manager → sort by CPU
  • Disable heavy startup programs
  • Update drivers / avoid thermal throttling
  • For servers: tune quantum, affinity, cgroups (Linux)

5. Link to this project

When CPU is saturated, waiting time for background jobs grows. Compare algorithms in the Simulator to see how different policies change avg wait and utilization.

6. Memory pressure

When RAM is full, the OS swaps data to disk — CPU looks busy but work slows down. Free memory before blaming the scheduler alone.
